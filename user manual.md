1.	Introduction
Task Insight Document Management System (DMS) is a secure, web-based platform designed to help organizations manage, store, and share documents efficiently. The system provides:
-	Centralized Storage: All your documents in one secure location
-	Role-Based Access Control: Different permission levels for different users
-	Audit Trail: Complete tracking of all document activities
2.	Basic Information
2.1	System Requirements
To use Task Insight DMS, you need:
-	A modern web browser (Chrome, Firefox, Safari, or Edge)
-	Stable internet connection
-	Valid user credentials (email and password)

2.2	Accessing the System
Open your web browser and navigate to your organization's Task Insight URL ( https://taskinsight.my/ ). You will be presented with the login page.

6.8	File Versioning

Task Insight DMS automatically tracks file versions when you upload a new version of an existing file. This allows you to access and restore previous versions of your documents.

How Versioning Works:

Automatic Version Creation:
-	When you upload a file with the same name as an existing file, a new version is created
-	Previous versions are preserved automatically
-	Each version maintains its upload date, uploader information, and file size

Viewing File Versions:

Method 1: Using Context Menu
1. Right-click on a file
2. Select "Manage versions" from the context menu
3. The version history modal will open

Method 2: From File Information
1. Right-click on a file
2. Select "File information" from the context menu
3. Click on "Version History" tab

Version History Display:

The version history shows:
-	Version number (v1, v2, v3, etc.)
-	Upload date and time
-	Uploader name
-	File size for each version
-	Version notes (if added)

Version Actions:

View Version:
-	Click on any version to preview it
-	You can download any previous version

Restore Version:
1. Select the version you want to restore
2. Click "Restore this version"
3. Confirm the restoration
4. The selected version becomes the current version

Delete Version (Admin Only):
-	Organization admins can delete old versions to save storage space
-	Deleting a version is permanent and cannot be undone

Version Best Practices:
-	Review version history before restoring to ensure you're selecting the correct version
-	Important: Restoring a version replaces the current version, but previous versions remain available
-	Use version notes (if available) to document changes in each version

7.3	Trash & Recovery Rules

Task Insight DMS uses a trash system to safely manage deleted files and folders. Deleted items are moved to Trash and can be recovered within a specified period.

Accessing Trash:

To view deleted items:
1. Click "Trash" in the sidebar navigation
2. All deleted files and folders will be displayed
3. Deleted items show their original location and deletion date

Recovery Period:

-	Deleted files and folders remain in Trash for 30 days (default)
-	After 30 days, items are permanently deleted automatically
-	The recovery period may vary based on organization settings

Restoring Items from Trash:

Method 1: Restore to Original Location
1. Go to "Trash" in the sidebar
2. Find the file or folder you want to restore
3. Right-click on the item
4. Select "Restore" from the context menu
5. The item will be restored to its original location

Method 2: Restore to Different Location
1. Go to "Trash" in the sidebar
2. Right-click on the item
3. Select "Restore to..." (if available)
4. Choose the destination folder
5. Click "Restore"

Permanently Deleting Items:

If you want to permanently delete items from Trash before the 30-day period:

1. Go to "Trash" in the sidebar
2. Right-click on the item you want to delete permanently
3. Select "Delete Permanently" or "Empty Trash" (for all items)
4. Confirm the permanent deletion
5. Warning: Permanently deleted items cannot be recovered

Emptying Trash:

To delete all items in Trash at once:
1. Go to "Trash" in the sidebar
2. Click "Empty Trash" button (if available)
3. Confirm the action
4. All items in Trash will be permanently deleted

Trash Rules and Limitations:

-	Deleting a folder moves the entire folder and all its contents to Trash
-	Restoring a folder restores all its contents to their original locations
-	You cannot edit or modify files while they are in Trash
-	Files in Trash still count toward your storage quota until permanently deleted
-	Only the file owner or organization admin can permanently delete items from Trash

Best Practices:

-	Regularly review your Trash to free up storage space
-	Restore accidentally deleted items as soon as possible
-	Use permanent deletion only when you're certain you no longer need the files
-	Remember that permanently deleted items cannot be recovered

8.6	Recurring Reminders
When you complete a recurring reminder:
1. The current reminder is marked as complete
2. A new reminder is automatically created for the next occurrence
3. The new reminder appears in your To-Do Documents

**Example:**
-	You set a weekly reminder for "Review weekly reports" every Monday at 9 AM
-	When you complete it on Monday, a new reminder is created for next Monday
-	This continues until the recurrence end date (if set)

8.7	Reminder Best Practices
Tips for effective reminder usage:
1. Use descriptive titles: "Review Q4 budget proposal" is better than "Review file"
2. Set appropriate priorities: Reserve "High" for truly urgent items
3. Add context in notes: Include what action needs to be taken
4. Use recurring reminders: For regular tasks like monthly reports
5. Check daily: Review your To-Do Documents each morning
6. Complete or dismiss: Keep your list clean by acting on reminders

Common Use Cases:
-	Contract reviews: Set reminders before contract expiration dates
-	Report submissions: Never miss a deadline
-	Document approvals: Track pending approvals
-	Regular reviews: Monthly/quarterly document reviews
-	Follow-ups: Remind yourself to follow up on shared documents
